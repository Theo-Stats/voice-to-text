# 语音转文字项目 - 技术文档

## 项目概述

本项目是一个基于 Whisper Large-V3-Turbo 模型的本地语音转文字应用，专为 Apple Silicon (Mac mini M4) 优化，提供 Web 界面和命令行两种使用方式，支持中文语音的高准确率转录，并具备智能标点恢复功能。

---

## 一、项目开发历程

### 1.1 需求分析阶段

**原始需求：**
- 将本地录音文件转换为文字
- 通过 HTML 网页应用进行交互
- 使用 Whisper large-v3 模型保证准确率
- 支持多次重复使用

**硬件环境：**
- 设备：Mac mini M4
- 内存：16GB
- 存储：256GB
- 加速：Apple Silicon MPS

### 1.2 技术选型阶段

| 需求 | 候选方案 | 最终选择 | 选择理由 |
|------|----------|----------|----------|
| 语音识别模型 | Whisper large-v3 / large-v3-turbo | large-v3-turbo | 准确率接近 large-v3，速度提升 8 倍 |
| 后端框架 | Flask / FastAPI / Django | Flask | 轻量级，适合小型应用 |
| 前端技术 | 纯 HTML / React / Vue | 纯 HTML + JavaScript | 简单直接，无需构建 |
| 音频处理 | librosa / pydub | librosa | 与 Whisper 配合良好 |
| 标点恢复 | 规则方法 / CT-Transformer | CT-Transformer | 基于模型，效果更自然 |
| 繁简转换 | OpenCC / zhconv | OpenCC | 成熟稳定，转换准确 |

### 1.3 开发迭代过程

```
迭代 1: 基础转录功能
    ↓

    ↓
```

## 三、核心技术组件

### 3.1 语音识别模型 - Whisper

**模型选择：**
- 模型名称：`openai/whisper-large-v3-turbo`
- 参数量：约 8 亿参数
- 特点：在 large-v3 基础上优化，速度提升 8 倍

**为什么选择 turbo 版本：**
```
large-v3:      准确率最高，但速度较慢
large-v3-turbo: 准确率接近 large-v3，速度快 8 倍
```

**加速方案：**
```python
# Apple Silicon MPS 加速
if torch.backends.mps.is_available():
    device = "mps"
    model = model.to(device)
    input_features = input_features.half()  # 半精度
```

### 3.2 音频处理 - librosa

**核心功能：**
```python
# 加载音频并转换采样率
audio, sr = librosa.load(audio_path, sr=16000)

# 切片处理 (30秒一片)
chunk_samples = 30 * sr
for i in range(total_chunks):
    chunk = audio[i * chunk_samples : (i + 1) * chunk_samples]
```

**切片时长选择：**
| 切片时长 | 优点 | 缺点 |
|----------|------|------|
| 10秒 | 上下文准确 | 速度慢，模型调用次数多 |
| 30秒 | 平衡速度和准确率 | 推荐使用 |
| 60秒 | 速度快 | 可能丢失上下文 |

### 3.3 标点恢复 - CT-Transformer

**模型来源：**
- 项目：FunASR / CT-Transformer-punctuation
- 类型：基于 ONNX 的中文标点预测模型
- 特点：轻量级，无需 GPU，基于语义预测

**工作原理：**
```
输入: "今天天气很好我们出去玩吧"
      ↓
CT-Transformer 分析语义
      ↓
输出: "今天天气很好，我们出去玩吧。"
```

**集成方式：**
```python
from cttPunctuator import CttPunctuator
punctuator = CttPunctuator()
result = punctuator.punctuate(text)[0]
```

### 3.4 繁简转换 - OpenCC

**用途：**
- Whisper 可能输出繁体中文
- 统一转换为简体中文输出

```python
from opencc import OpenCC
cc = OpenCC('t2s')  # Traditional to Simplified
simplified = cc.convert(traditional_text)
```

## 六、关键技术决策

### 6.1 为什么选择本地运行而非 API？

| 方案 | 优点 | 缺点 |
|------|------|------|
| OpenAI API | 无需本地算力 | 按分钟收费，成本高 |
| 本地运行 | 免费，隐私安全 | 需要本地算力 |

**决策：** 用户有 Mac mini M4 (16GB)，足够运行 large-v3-turbo，选择本地运行。

### 6.2 为什么选择 30 秒切片？

```
切片太短 (10秒):
  - 模型调用次数多
  - 总耗时长
  - 但上下文更准确

切片太长 (60秒):
  - 模型调用次数少
  - 总耗时短
  - 但可能丢失边界上下文

30秒切片:
  - 平衡速度和准确率
  - 业界常用配置
```

### 6.3 为什么需要标点恢复？

```
Whisper 输出特点:
  - 中文转录通常不带标点
  - 输出: "今天天气很好我们出去玩吧"

用户需求:
  - 需要可读性好的文本
  - 需要: "今天天气很好，我们出去玩吧。"

解决方案:
  - CT-Transformer 基于语义预测标点
  - 效果自然，无需规则维护
```

## 九、已知限制与未来优化

### 9.1 当前限制

1. **内存占用：** large-v3-turbo 需要约 3-4GB 内存
2. **处理速度：** 1分钟音频约需 10-20 秒处理
3. **标点准确率：** CT-Transformer 在专业术语场景可能不准确

### 9.2 未来优化方向

1. **实时转录：** 支持麦克风实时输入
2. **多语言支持：** 扩展到英文、日文等
3. **说话人分离：** 区分不同说话人
4. **字幕生成：** 直接生成 SRT 字幕文件
